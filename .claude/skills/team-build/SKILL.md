# 팀 빌드 모드 (리드 + 구현 서브에이전트)

여러 화면·기능을 서브에이전트 팀으로 병렬 구현하는 모드. 리드가 분해·검수하고, 구현은 저비용 모델부터 승격시킨다.

## 역할

- **리드 = 현재 세션**: 작업 분해, 배치 설계, 서브에이전트 프롬프트 작성, 결과 검수, 최종 리뷰. 원칙적으로 직접 구현하지 않음 (foundation/공용 유틸·설치 등 작고 전역적인 것만 예외)
- **구현 = Agent 서브에이전트**: 기본 `model: "sonnet"`, `subagent_type: "general-purpose"`

## 에스컬레이션 (Sonnet → Opus → Fable/리드)

1. Sonnet 결과가 검수 불합격(요구 미충족, `npm run check` 실패, CLAUDE.md 규칙 위반)이면 **구체적 피드백을 담아 같은 에이전트에 SendMessage로 1회 재시도**
2. 재시도 실패, 또는 애초에 어려운 태스크(서버 보안/RLS 로직, 복잡한 상태머신, 마이그레이션, 결제·주문 금액 로직)는 처음부터/승격해서 `model: "opus"`
3. Opus도 막히면 리드가 직접 구현 (리드 모델이 fable이면 그게 최종 티어)

## 진행 규칙

- **Foundation-first**: 라이브러리 설치, 공용 컴포넌트/유틸(토스트, 포맷터, 상수)을 첫 배치로 끝낸 뒤 화면 배치를 병렬로. 화면 에이전트가 공용물을 각자 재발명하지 않게 함
- **파일 소유권 분리**: 같은 파일을 두 에이전트가 동시에 만지지 않도록 배치를 나눔. 겹치면 순차로
- 독립 배치는 **한 메시지에서 병렬 스폰**
- 서브에이전트 프롬프트에 반드시 포함:
  - 대상 파일 경로(정확히), 참조할 시안/스펙 문서 경로
  - CLAUDE.md 절대 규칙 중 해당 태스크 관련 항목 요약
  - 디자인 토큰은 `docs/design-system.md` 기준, 이모지 아이콘 금지(라인 SVG)
  - 완료 기준: `npm run check` 통과, 기존 기능 회귀 없음
  - **임의 범위 확장 금지** (시키지 않은 파일 수정 금지)
- **검수 게이트**: 모든 서브에이전트 결과는 리드가 diff 확인 + `npm run check` 실행 후 합격 판정. 불합격 시 에스컬레이션 절차. 의존 배치는 선행 배치 합격 전 시작 금지

## 마감 (전 배치 완료 후)

1. `npm run check` + `npm run lint` + `npx vitest run` 전체 통과
2. `/code-review` (medium 이상) → 지적사항 수정
3. 런타임 검증 필요 시 `/verify` 또는 dev 서버 스모크
4. 필수 테스트 대상 로직(주문 재계산, 품절/전화문의 차단, 장바구니 병합, RLS)을 건드렸으면 테스트 없이 커밋 금지
5. 커밋은 사용자 확인 후. 멀티라인 커밋 메시지는 `git commit -F <file>` 사용
