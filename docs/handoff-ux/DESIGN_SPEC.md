# DESIGN_SPEC — UX 보강 4종 ([9]~[12])

기존 디자인 시스템(색·타이포·헤더/GNB/푸터/모바일 하단바)을 그대로 재사용합니다.
공통 크롬(헤더, 네이비 GNB, 옐로 CTA, 다크 푸터, 모바일 하단 고정바 4칸, 드로어)은 기존 시안과 동일 — 여기서는 **새/보강된 부분**만 상세히 기술합니다.

접근성 공통: 흰 배경 텍스트 대비 4.5:1↑, 페이지당 h1 1개, 아이콘은 라인 SVG(이모지 금지), 터치 타깃 44px↑, `lang="ko"`.

## [9] 상태·피드백 화면 세트 — `탄탄편의시설-상태피드백.dc.html`

미리보기 스위처(하단 중앙, 시안용): 주문완료 / 문의조회 / 404 / 검색0건 / 토스트. 실제로는 각 라우트로 분리.

### (1) 주문완료 (무통장 강화판) — `/order/complete/[orderNo]`
- 상단 중앙: 초록 원형 체크 아이콘(배경 `#E2F1E8`, stroke `#1B6B3A`, `ttpop` 팝 애니메이션 0.4s) + "주문이 정상 접수되었습니다" + 주문번호(모노스페이스 `#1F3864`).
- **입금 안내 카드(화면 주인공)**: 네이비 2px 보더, 헤더 바(네이비 배경 + 옐로 카드 아이콘 + "무통장 입금 안내").
  - 입금 금액: 40px/900 네이비, 중앙 정렬 (가장 큰 요소).
  - 정보 리스트(보더 박스): 입금 은행 / 계좌번호 + **복사 버튼** / 예금주 / 입금자명.
  - 계좌 복사 버튼: 아웃라인(네이비), 클릭 시 `navigator.clipboard.writeText(계좌번호숫자)` + **성공 토스트** "계좌번호가 복사되었습니다".
  - 안내 박스(옐로 틴트 `#FBF1D8`, 텍스트 `#7a5a00`): "입금 확인 후 배송 준비가 시작됩니다. 주문 후 24시간 이내 입금…".
- 하단 버튼 2열: `주문 상세 보기`(네이비 솔리드) / `쇼핑 계속하기`(네이비 아웃라인). 그 아래 입금 문의 전화 링크.

### (2) 비회원 문의 조회 — `/inquiry/check`
- 중앙 카드(최대 560px). 안내 문구 "문의 작성 시 입력하신 이름과 비밀번호를…".
- 폼: 이름 / 비밀번호(type=password) + `문의 조회하기`(네이비 솔리드, 풀폭). 비밀번호 분실 시 전화 안내.
- **조회 성공 상태**: 폼 → 결과 카드로 전환. 상단 세그먼트 토글로 두 변형 확인:
  - **답변 완료**: 상태 배지 `답변 완료`(`#E2F1E8/#1B6B3A`), 문의 제목/작성일/내용 + **관리자 답변 블록**(네이비 "탄탄" 아바타 + 답변 본문, 배경 `#fbfcfe`).
  - **답변 대기**: 배지 `답변 대기`(`#FBF1D8/#8F6A00`), 답변 블록 대신 시계 아이콘 + "답변을 준비하고 있습니다. 영업시간(평일 09~18시)…".
  - `다른 문의 조회하기`로 폼 복귀.
- 서버 구현 메모: RLS상 비회원은 본인 글 조회 불가 → **서버 라우트에서 scrypt 비밀번호 검증 후** 문의+답변 반환.

### (3) 404 / 에러 — `+error.svelte` (404 등)
- 대형 `404`(96px/900 네이비, 밑줄 옐로 하이라이트 바). "페이지를 찾을 수 없습니다" + 부드러운 안내.
- 버튼 2개: `홈으로 가기`(네이비 솔리드, home 아이콘) / `{전화번호} 전화하기`(옐로, tel:). **중장년 이탈 방지 위해 전화 동선 강조**. 상담 시간 문구.

### (4) 검색 결과 0건 (상품 리스트 내 상태)
- 브레드크럼 + "'{검색어}' 검색 결과" + "총 0개" + 카테고리 칩.
- 빈 상태 카드: 돋보기 원형 아이콘 + "'{검색어}' 검색 결과가 없습니다" + "찾으시는 상품이 없나요? 전화 주시면 … 찾아드립니다".
  - 버튼: `{전화번호} 전화 문의`(옐로) / `전체 상품 보기`(아웃라인). + 추천 검색어 칩.

### (5) 토스트 컴포넌트 (전역)
- **성공**: 배경 네이비 `#1F3864`, 흰 텍스트, 옐로 원형 + 네이비 체크 아이콘.
- **오류**: 배경 `#A3403A`, 흰 텍스트, 흰 원형 + 빨강 X 아이콘.
- 위치: **PC 우하단(right:24 bottom:24)**, **모바일 하단 중앙(좌우 16px stretch)**. 진입 `ttslide` 0.28s, **약 2.6초 후 자동 소멸**. 여러 개 stack (세로 gap 10px).
- 라이브러리: 코드베이스는 **svelte-sonner** 사용 예정 — 위 스타일을 커스텀 테마로 적용.

## [10] 마이페이지 주문 상세 + 상태 타임라인 — `탄탄편의시설-마이페이지-주문상세.dc.html`  `/my/orders/[orderNo]`

- 상단: 브레드크럼, 주문번호/주문일, 우측 현재 상태 배지.
- **상태 타임라인(핵심)**: 5단계 `입금대기 → 입금확인 → 배송준비 → 배송중 → 배송완료`.
  - PC: 가로 스텝퍼. 연결선(`#E6E9EE`) 위에 진행률만큼 네이비 라인(width = `현재index / 4 * 80%`).
  - 완료 단계: 네이비 원 + 흰 체크. 현재 단계: 네이비 원 + **옐로 도트** + 옐로 링(box-shadow `0 0 0 4px rgba(242,183,5,.2)`) + 네이비 굵은 라벨. 이후 단계: 회색 원/회색 도트/회색 라벨.
  - 모바일(≤820px): 세로 스텝퍼(도트 + 세로 연결선).
- **배송 조회**(현재 단계가 배송중 이상일 때만 노출): 택배 아이콘 + `택배사명 + 송장번호` + `배송 조회` 링크 버튼(택배사 조회 URL 새 탭).
  - 택배사 조회 URL 매핑 상수화(CJ/롯데/한진/우체국/로젠). CJ 예: `https://trace.cjlogistics.com/next/tracking.html?wblNo={송장}`.
- **주문 상품 스냅샷**: 썸네일/상품명/옵션/수량/금액 (주문 시점 값 저장).
- **배송지 정보**: 받는분/연락처/주소/요청사항 (라벨-값 그리드).
- **결제 정보**(우측 sticky): 결제수단/상품금액/배송비(착불), 입금 계좌 박스(은행·계좌·예금주·입금자명), 총 결제금액(24px 네이비).
- **취소 동선 변형 2종(상태 기반)**:
  - 취소 가능(입금대기·입금확인): `주문 취소 요청`(아웃라인 `#A3403A`) → **확인 다이얼로그**("주문을 취소하시겠습니까?" + 환불 안내 + 돌아가기/취소 요청) → 접수 토스트.
  - 배송준비 이후: 버튼 대신 옐로 틴트 안내 "배송 준비가 시작되어 온라인 취소가 어렵습니다. 취소는 전화로 문의해 주세요." + 전화 버튼.

## [11] 상품 상세 보강 — `탄탄편의시설-상품상세-보강.dc.html`  `/products/[id]`

기존 [3] 상품상세 **레이아웃 유지**, 아래 2가지만 추가.

### (1) 이미지 라이트박스
- 대표 이미지에 "이미지 크게 보기" 힌트 배지(우상단, 반투명 다크). 이미지/힌트 클릭 시 오픈. 갤러리 썸네일 5장.
- 오버레이(`rgba(10,14,22,.94)`, z-index 위): 상단 바(상품명 + `현재/전체` 인디케이터 예 `3 / 5` + 닫기 버튼 44px), 중앙 큰 이미지, **좌/우 이동 화살표**(원형, prev/next 순환), 하단 **썸네일 스트립**(현재 항목 옐로 보더/불투명, 나머지 반투명).
- 모바일(≤820px): 화살표 축소 + **스와이프 힌트** "← 좌우로 밀어 사진을 넘겨보세요 →". 실제 구현 시 터치 스와이프 + 핀치줌(native `<dialog>` 권장).

### (2) 전화문의 이중 동선 (전화문의 상품 구매 패널)
- 주(primary): `전화 문의하기 {전화번호}`(옐로 솔리드, tel:).
- 부(secondary): 그 아래 `문의 남기기`(네이비 아웃라인) → `/inquiry?type=quote&product={상품명}` 프리필.
- 보조 문구(회색): "영업시간(평일 09~18시) 외에는 문의를 남겨주세요. 확인 후 순차 연락드립니다."
- 시각 위계: 전화=채움, 문의=아웃라인. 모바일 하단 구매바에도 `문의`(좁게) + `전화 문의하기`(넓게) 2버튼.

## [12] 관리자 보강 — `탄탄편의시설-관리자-보강.dc.html`

60대 대표 사용 — 버튼 44px↑, 한 화면 한 작업.

### (1) 주문 목록 원클릭 입금확인 — `/admin/orders`
- 관리 칸: **결제대기(입금대기) 상태 행에만** 인라인 `입금확인` 버튼(옐로, min-height 44px, 체크 아이콘) + 기존 `상세` 버튼.
- 클릭 → **확인 다이얼로그**: "입금확인 처리할까요? / 주문 {주문번호} 을(를) 입금확인 처리합니다. / 입금 금액 {금액}(30px 네이비)" + `돌아가기`/`입금확인`(옐로).
- 확인 → 해당 주문 상태 `결제완료`로 변경(상태 탭 카운트 즉시 반영) + **성공 토스트** "입금확인 처리되었습니다"(우하단).
- 상태 3종: 대기(버튼) / 확인 다이얼로그 / 처리완료(토스트).

### (2) 배너 등록 폼 — `/admin/site/banners`
- 업로드 영역: 안내 "권장 **1920 × 600px** · 5MB 이하 · JPG / PNG". (클라이언트 압축 browser-image-compression 권장.)
- 업로드 직후 **크롭 미리보기 2종**(실제 배너 비율):
  - **PC 프레임**: 브라우저 크롬 목업 + 1920×600 비율 영역, "배너 전체 노출 (크롭 없음)".
  - **모바일 프레임**: 폰 목업 + 중앙 크롭 영역(상하 잘림 점선 표시), "중앙 크롭 영역" + "모바일에서는 상하 일부가 잘립니다. 핵심 문구는 가운데 배치".
  - 파일명/용량 + `다시 업로드` 버튼.
- 입력: 연결 링크 / 노출 시작일 / 노출 종료일 / **노출 순서**(number). `배너 등록하기`(네이비 솔리드, 44px↑).
- 기존 배너 목록: 드래그 핸들(순서 변경) + 미리보기 + 제목/링크/노출기간 + 노출/숨김 토글.

## 디자인 토큰 요약
- 색: `#1F3864` `#141b2b` `#F2B705` `#FFC933` `#F7F8FA` `#eef1f5` `#8F6A00` `#6b7488` `#E6E9EE` `#eef2f8`
- 상태색: 성공 `#1B6B3A`/`#E2F1E8` · 경고 `#8F6A00`/`#FBF1D8` · 오류 `#A3403A`/`#FBF1F1` · 정보 `#234F9E`/`#DDE8FF`
- 라운드: 카드 12~14px, 버튼 9~11px, 배지/칩 6~20px. 섀도: 카드 `0 4~10px rgba(20,27,43,.05~.09)`, 토스트 `0 8px 22px`.
- 타이포: Pretendard. 본문 15~16px, 라벨 13.5~15px/800, 금액/타이틀 24~40px/900.
- 애니메이션: `ttpop`(체크 팝 0.4s), `ttslide`(토스트 진입 0.28s), 타임라인 진행 `width .3s`.

## 상태 관리 요약
- 주문완료: 서버에서 주문/입금 정보 로드. 계좌 복사 = clipboard + 토스트.
- 문의조회: 서버 라우트 scrypt 검증 → {문의, 답변, done}. done에 따라 답변완료/대기 렌더.
- 주문상세: `order.status`(5단계) → 타임라인 index + 취소버튼/전화안내 분기 + 배송조회 노출.
- 관리자 입금확인: `dialog(orderId)` / 처리 시 status='결제완료' + `toast`.
- 배너: `uploaded` bool → dropzone/preview 전환.
- 전역 토스트: svelte-sonner, 성공(네이비)/오류(적색) 테마.

## 에셋
- 실제 이미지 없음 — 모든 이미지 자리는 줄무늬 플레이스홀더 + 모노스페이스 설명. 실제 상품/배너/시공 사진으로 교체 필요.
- 아이콘: 인라인 라인 SVG(stroke 1.8~2.4px). 아이콘 라이브러리(lucide 등)로 대체 가능.
