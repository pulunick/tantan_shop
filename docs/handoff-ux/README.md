# 탄탄 편의시설 — UX 보강 핸드오프 패키지 ([9]~[12])

이 폴더는 기존 쇼핑몰 디자인에 이어 만든 **상태·피드백 화면 + 기능 보강 4종**을
로컬 **Claude Code**로 실제 구현할 때 그대로 전달하는 묶음입니다.
기존 디자인 시스템(`handoff/DESIGN_SPEC.md`)과 `docs/ux-enhancements.md` 스펙을 따릅니다.

---

## 0. 전달 방법

1. 이 프로젝트 우측 상단 **다운로드**로 `handoff-ux/` 폴더(zip)를 내려받기
2. 로컬에서 zip 풀기 → 기존 프로젝트 폴더에 함께 두기 (예: `~/projects/tantan-shop/`)
3. 그 폴더에서 터미널 열고 `claude` 실행
4. Claude Code에게 첫 지시:
   > "`handoff-ux/README.md`와 `handoff-ux/DESIGN_SPEC.md`를 먼저 읽어. `designs/*.dc.html`이 완성된 디자인 시안이야. 기존 코드베이스(SvelteKit + Supabase)의 패턴/디자인 토큰에 맞춰 이 화면들을 구현해줘."

**완성된 디자인 시안 (`designs/*.dc.html` — 브라우저에서 바로 열림):**

| 파일 | 화면 | 프롬프트 |
|---|---|---|
| `탄탄편의시설-상태피드백.dc.html` | 주문완료(무통장 강화) · 비회원 문의조회 · 404 · 검색 0건 · 토스트 컴포넌트 | [9] |

## 2. `.dc.html`을 어떻게 읽나

디자인 **시안(레퍼런스)**입니다. 런타임(`support.js`)을 그대로 쓰지 말고,
마크업의 인라인 스타일 값(hex·여백·폰트)과 하단 `<script data-dc-script>` 로직 클래스의
상태/핸들러를 **기존 스택 컴포넌트로 재현**하세요.

> "`.dc.html`은 커스텀 프리뷰 포맷이야. 마크업의 인라인 스타일과 로직 클래스의 상태·동작을 그대로 Svelte 컴포넌트로 옮기고, 미리보기용 상태 스위처는 제거해."
