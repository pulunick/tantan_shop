# Handoff: 마이페이지 섹션 리디자인

## Overview
탄탄 편의시설 쇼핑몰의 **마이페이지 섹션 전체** 정식 디자인입니다. 기존 `src/routes/(my)/mypage/` 하위 화면들(개요·주문내역·배송지·정보수정·내 문의·회원탈퇴)을 쇼핑몰 본편과 동일한 디자인 시스템으로 통일하고, 그동안 없던 **쇼핑몰 공통 헤더/GNB/푸터/모바일 하단 바**를 마이페이지에도 적용해 쇼핑 동선을 연결하는 것이 핵심입니다.

주문 상세 화면은 이미 확정된 **[10] 타임라인 시안**을 그대로 유지합니다(재디자인 금지). 나머지 화면이 그 톤에 맞춰 정렬되어 있습니다.

## About the Design Files
이 번들의 파일은 **HTML로 만든 디자인 레퍼런스**입니다 — 최종 외형과 동작을 보여주는 프로토타입이며, 그대로 복사해 배포하는 프로덕션 코드가 아닙니다.

작업 목표는 이 HTML 디자인을 **대상 코드베이스(SvelteKit + Tailwind CSS v4)의 기존 환경과 패턴으로 재현**하는 것입니다. 이 프로젝트에는 이미 마이페이지 구현이 존재하므로, 아래 매핑 표를 따라 **기존 라우트/컴포넌트를 리디자인**하면 됩니다.

- `.dc.html` 파일은 브라우저에서 바로 열립니다. `support.js`는 프로토타입 렌더 런타임일 뿐이며, **프로덕션 코드에는 포함하지 마세요.**
- 상태 전환(섹션 스위칭, 모달, 토스트 등)은 데모를 위한 클라이언트 로직입니다. 실제 구현은 SvelteKit 라우트 분리 + form actions로 대체합니다(아래 참조).

## Fidelity
**High-fidelity (hifi).** 최종 컬러·타이포·간격·인터랙션이 확정된 픽셀 단위 시안입니다. 기존 코드베이스의 Tailwind 토큰과 컴포넌트로 픽셀에 가깝게 재현하세요. 임의의 새 색상·폰트를 도입하지 마세요.

## 기존 코드베이스 매핑 (가장 중요)

| 디자인 섹션 (마이페이지.dc.html 내 패널) | 대상 라우트 파일 | 비고 |
| --- | --- | --- |
| 공통 셸(헤더/GNB/푸터/모바일바/드로어) + 좌측 메뉴 | `src/routes/(my)/+layout.svelte` | 현재 (shop) 그룹 전용이던 공통 헤더/푸터를 (my)에도 적용. 브레드크럼·"쇼핑 계속하기" 추가 |
| 요약 카드("OO님 안녕하세요" + 진행중 주문 배지) | `src/routes/(my)/mypage/+page.svelte` | 개요 페이지 상단. `data.profileName`, 진행중 주문 카운트 사용 |
| 주문내역 목록 + 빈 상태 | `src/routes/(my)/mypage/orders/+page.svelte` | `data.orders`, `StatusPill` 재사용 |
| 배송지 관리 + 빈 상태 | `src/routes/(my)/mypage/addresses/+page.svelte` | ⚠ 아래 "배송지" 주의 참조 |
| 회원정보 수정 + 비밀번호 변경 | `src/routes/(my)/mypage/profile/+page.svelte` | 비밀번호 변경 카드 신규 추가 |
| 내 문의 목록/답변 펼침 + 빈 상태 | `src/routes/(my)/mypage/inquiries/+page.svelte` | `data.inquiries` |
| 회원탈퇴 | `src/routes/(my)/mypage/withdraw/+page.svelte` | 확인 입력("탈퇴") + 동의 체크 후 활성화 |
| 주문 상세 (확정 시안, 유지) | `src/routes/(my)/mypage/orders/[id]/+page.svelte` | 재디자인 금지. 타임라인 시안 그대로 |

좌측 메뉴 항목 순서와 활성 상태 로직은 이미 `+layout.svelte`의 `navItems` / `isActive()`에 존재합니다 — 스타일만 아래 스펙으로 교체하세요.

## Screens / Views

### 공통 셸
- **헤더**: 상단 유틸바(navy-dark, 로그인 상태에서 "김현동 님 / 로그아웃 / 마이페이지 / 고객센터"), 로고, 검색바(PC), 대표번호 CTA(yellow), 장바구니 아이콘. **(shop) 그룹의 기존 헤더 컴포넌트를 (my)에서도 렌더** — 지금은 없어서 쇼핑 복귀 동선이 끊겨 있음.
- **GNB**: navy 배경, 상품/시공사례/회사소개/고객센터 + 우측 시공·견적 문의 CTA.
- **브레드크럼**: `홈 › 마이페이지` (왼쪽) + `쇼핑 계속하기` outline 버튼(오른쪽, `/products`로 이동, `‹` 아이콘).
- **푸터**: navy-dark, 회사정보/바로가기/무통장 계좌 3열. 모바일에서 1열.
- **모바일 하단 고정 바** (max-width 820px에서만): 4칸 — 전화하기(yellow) / 메뉴(드로어) / **내정보(활성, navy-tint)** / 장바구니. height 66px, body padding-bottom 66px. **이 '내정보' 탭이 마이페이지 진입점** — 모바일 우선.
- **드로어**: 좌측 슬라이드. 상단 사용자 정보(navy-tint) + 메뉴 링크 + 로그아웃 + 하단 시공·견적 CTA.

### 레이아웃
- 데스크톱: `max-width 1200px` 컨테이너, `grid-template-columns: 230px 1fr`, gap 24px, 좌측 메뉴 `position: sticky; top:16px`.
- 모바일(≤820px): 좌측 메뉴 숨김 → **상단 가로 스크롤 탭**(주문내역/배송지 관리/회원정보 수정/내 문의/회원탈퇴, pill 형태, 활성=navy 솔리드). 본문 1열.

### 좌측 메뉴 (PC)
- 상단: 원형 아바타 아이콘(navy-tint) + 이름/이메일.
- 항목: 주문내역 / 배송지 관리 / 회원정보 수정 / 내 문의. 활성 항목은 좌측 3px yellow 보더 + navy-tint 배경 + navy 800. 우측에 개수 배지(주문내역·내 문의).
- 구분선 아래: 로그아웃 링크.
- 하단: 작게 밑줄 텍스트 "회원탈퇴"(#A7ADB9, 12.5px) — 실수 클릭 방지용 절제.

### 요약 카드 (개요 상단)
- navy 배경 카드. 왼쪽: "김현동 님, 안녕하세요"(h1, 흰색) + 감사 문구(#B9C4DC).
- 오른쪽: 반투명 칩 안에 "진행 중 주문 / 배송·준비 중" 라벨 + yellow 카운트 배지(진행중 = pending/paid/preparing/shipping 개수).

### 주문내역
- 카드 리스트. 각 카드: 상단 스트립(bg, 하단 보더) = `주문번호 TT-… · 날짜`(좌) + 상태 배지(우). 본문 = 썸네일 플레이스홀더(66px, 45° 스트라이프) + 상품 요약 + "총 N개 상품" + 금액(19px/900) + **상세보기** outline 버튼(→ 주문 상세, 44px+).
- 빈 상태: 카트 라인 아이콘, "주문 내역이 없습니다." + 안내 + "상품 보러가기" navy 솔리드 버튼(→ `/products`).

### 배송지 관리 — ⚠ 주의
현재 `addresses/+page.svelte` 주석에 따르면 **Phase 1엔 배송지 테이블이 없어 읽기전용**입니다. 이 시안은 **추가/수정/삭제/기본설정이 있는 완성형 CRUD UI**를 제시합니다. 구현 시 둘 중 택1:
1. 배송지 테이블 도입 후 CRUD form actions 구현(시안대로), 또는
2. Phase 1 유지 시 카드/빈 상태 스타일만 시안 톤으로 맞추고 액션 버튼은 후속 단계로.
- 카드: 이름·연락처·기본배송지 배지(navy-tint) + 주소. 하단 액션: 수정(outline) / 삭제(빨강 outline) / 기본으로 설정(비기본 카드만). **기본 배송지 카드는 보더가 navy.**
- 추가/수정은 모달(받는분·연락처·우편번호+주소검색·기본주소·상세주소·기본배송지 체크). 삭제는 확인 다이얼로그.
- 빈 상태: 핀 아이콘, "저장된 배송지가 없습니다." + "배송지 추가" 버튼.

### 회원정보 수정
- 카드 1 "기본 정보": 이메일(읽기전용, bg 배경/sub 텍스트 + "변경할 수 없습니다" 안내), 이름·휴대폰 입력, "기본 정보 저장" navy 솔리드 버튼. → 기존 `profile` form action(`?/` default) 유지.
- 카드 2 "비밀번호 변경"(신규, 분리 카드): 현재/새/새 확인 3개 password 입력 + "비밀번호 변경" outline 버튼. → 별도 form action 추가 필요(예: `?/password`).
- 최대 너비 560px.

### 내 문의
- 카드 리스트. 상단: 유형 배지(navy-tint) + 답변완료/답변대기 배지 + 날짜(우). 제목(16px/800). 본문 미리보기(sub).
- 답변완료 항목은 클릭 시 **답변 펼침**(캐럿 회전). 답변 블록: bg 배경, "탄탄 편의시설 답변" 라벨 + 체크 아이콘 + 답변일 + 답변 본문(white-space: pre-line).
- 우상단 "문의하기" outline 버튼(→ `/inquiry` 또는 고객센터).
- 빈 상태: 말풍선 아이콘, "문의 내역이 없습니다." + "문의하기" 버튼.

### 회원탈퇴
- 최대 너비 600px 카드. 상단 경고 박스(#FBF1F1 배경/보더): "탈퇴 전 꼭 확인해 주세요" + 3개 안내(개인화 데이터 이용 불가 / **주문·결제 기록은 법령에 따라 보존** / 복구 불가).
- 확인 입력란에 "탈퇴" 정확히 입력 + 동의 체크박스(accent #A3403A) **둘 다 충족 시에만** 탈퇴 버튼 활성.
- 탈퇴 버튼: **절제된 outline 스타일**(비활성 시 회색, 활성 시 #A3403A 보더/텍스트). 클릭 시 최종 확인 다이얼로그. → 기존 `withdraw` `?/withdraw` action 연결(주문 이력 있으면 익명화 탈퇴).

## Interactions & Behavior
- **섹션 스위칭**: 프로토타입은 단일 파일에서 상태로 전환하지만, **실제 구현은 기존처럼 라우트 분리**(`/mypage/orders` 등)로 하고 `+layout.svelte`가 active 상태를 표시합니다.
- 폼 제출: SvelteKit `use:enhance` + form actions. 성공/실패 메시지는 기존 profile/withdraw 패턴(상태 배너) 사용. 프로토타입의 토스트는 참고용.
- 문의 답변 펼침: 답변완료 항목만 토글 가능, 캐럿 180° 회전 transition.
- 배송지 기본설정: 하나 지정 시 나머지 해제, 목록에 항상 기본 1개 유지.
- 반응형: 820px 브레이크포인트. 이하에서 상단 유틸바/PC 검색/GNB/좌측 메뉴 숨김, 모바일 검색 아이콘·햄버거·하단바·상단 탭 노출.
- 접근성: 페이지당 h1 1개(요약 인사말), 아이콘 버튼 aria-label, 포커스 보더 유지, 터치 타깃 44px+, 텍스트 대비 4.5:1+.

## Assets
- 폰트: Pretendard (variable) — 기존 프로젝트 설정 사용.
- 아이콘: 인라인 라인 SVG (stroke #1F3864, 1.8px). 이모지 금지. 코드베이스의 기존 아이콘 세트로 대체하세요.
- 상품 이미지: 45° 스트라이프 플레이스홀더로 표시됨 — 실제 상품 썸네일로 교체.
